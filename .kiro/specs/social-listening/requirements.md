# Requirements Document

## Introduction

The Vietnamese Social Listening & Reputation Protection Platform is a production-ready MVP system designed to monitor public online sources for brand mentions, detect negative sentiment and misinformation risks, generate alerts for crisis situations, and support legal compliance workflows for reputation management.

**Core Workflow:**
User creates keyword group → user adds sources → system scans sources (manual or scheduled) → system stores mentions → AI analyzes sentiment and risk → system creates alert if risky → user creates incident → system supports evidence, response draft, takedown request draft, report → dashboard shows the whole situation.

**Keyword → Source → Crawl → Mention → AI Analysis → Alert → Incident → Report → Legal Takedown**

The system monitors ONLY legally accessible public sources: Facebook public pages/groups/posts/comments, YouTube public videos/comments, RSS feeds, news websites, public web links, manually added URLs, and custom source groups. It analyzes content using AI to assess sentiment and risk levels, creates incident tickets for high-risk mentions, and generates comprehensive reports for stakeholders.

## Glossary

- **Platform**: The Vietnamese Social Listening & Reputation Protection Platform system
- **User**: An authenticated person using the Platform with assigned roles and permissions
- **Admin**: A User with administrative privileges to manage system configuration
- **Keyword**: A search term or phrase used to identify relevant mentions
- **Keyword_Group**: A collection of Keywords with logical operators (AND/OR/NOT), priority settings, and alert thresholds
- **Negative_Keyword**: A term that excludes mentions from results when matched
- **Excluded_Keyword**: A term that filters out irrelevant mentions from results
- **Source**: A public data origin (Facebook public page/group, YouTube public video/channel, RSS feed, news website, public web link, or manually added URL)
- **Source_Group**: A collection of Sources organized for management purposes
- **Mention**: A collected piece of content (article, post, comment) containing matched Keywords with metadata (title, content, URL, source name, platform type, published time, detected time, matched keywords, screenshot/evidence metadata, engagement metrics, raw metadata)
- **Crawl_Job**: A scheduled or manual task that scans Sources for Mentions
- **Scan_Schedule**: A configured time pattern for automatic Crawl_Job execution (manual + automatic scheduled)
- **AI_Analysis**: Automated sentiment, risk, and crisis assessment of a Mention using pluggable AI providers (OpenAI, Gemini, DeepSeek, Claude, local models)
- **Sentiment**: A classification of Mention tone (positive, neutral, negative with low/medium/high intensity)
- **Risk_Score**: A numerical value (0-100) indicating potential harm to brand reputation
- **Crisis_Level**: A severity classification (1-5) indicating urgency of response needed
- **Vietnamese_Summary**: A Vietnamese-language summary of the Mention generated by AI
- **Suggested_Action**: AI-recommended action (monitor, respond, escalate, legal_review)
- **Suggested_Department**: AI-identified responsible department (customer_service, PR, legal, executive)
- **Confidence_Score**: A numerical value (0-100) indicating AI analysis confidence
- **Alert**: A notification triggered when a Mention meets defined risk or sentiment thresholds
- **Alert_Threshold**: A Risk_Score value configured per Keyword_Group that triggers Alert creation
- **Notification_Channel**: A delivery method for Alerts (dashboard, email, Telegram, webhook, Zalo-ready)
- **Incident**: A tracked case created from a high-risk Mention requiring investigation and response
- **Incident_Owner**: The User assigned to manage an Incident
- **Incident_Status**: The current state of an Incident (new, verifying, responding, waiting_legal, waiting_platform, resolved, closed)
- **Incident_Deadline**: A target date for Incident resolution
- **Incident_Timeline**: A chronological record of status changes and actions for an Incident
- **Evidence_File**: A captured screenshot, URL archive, or metadata file attached to an Incident
- **Evidence_Metadata**: Structured data about captured evidence (URL, timestamp, hash, capture method)
- **Response_Draft**: A pre-written draft for public statements, private customer replies, corrections, legal takedowns, or press releases
- **Response_Type**: The category of response (public, private_customer, correction, legal_takedown, press)
- **Takedown_Request**: A formal request to remove or correct harmful content on a platform
- **Platform_Report_Checklist**: A structured form for reporting content to social media platforms (Facebook, YouTube, etc.)
- **Legal_Escalation_Checklist**: A structured form for escalating to Vietnamese authorities (Bộ TT&TT, Công an)
- **Human_Approval**: Required confirmation before external action (platform report, legal escalation, public response)
- **Report**: A generated summary document (daily, weekly, monthly, or crisis) of monitoring activity
- **Report_Type**: The category of report (daily, weekly, monthly, crisis, keyword_performance, negative_mention, source_risk)
- **Export_Format**: The output format for reports (PDF, JSON, CSV)
- **Dashboard**: A real-time visualization showing total mentions, negative mentions, crisis alerts, open incidents, sentiment chart, mentions over time, top risky keywords/sources/mentions, SLA overdue incidents
- **SLA_Overdue**: An Incident that has exceeded its deadline
- **Engagement_Metrics**: Quantitative data about content interaction (likes, shares, comments, views)
- **Spread_Risk**: A measure of content's potential to reach large audiences based on sharing velocity
- **Deduplication**: The process of identifying and removing duplicate Mentions
- **Rate_Limiting**: Controlled request frequency to respect Source server capacity and robots.txt directives
- **Robots_Txt**: A standard file that specifies crawling permissions for websites
- **Audit_Log**: A system record of User actions for compliance and security tracking
- **Pluggable_AI_Provider**: An AI service configured via environment variables (dummy, OpenAI, Gemini, DeepSeek, Claude)

## Requirements

### Requirement 1: User Authentication and Authorization

**User Story:** As a system administrator, I want to control who can access the Platform and what actions they can perform, so that sensitive monitoring data and operations are protected.

#### Acceptance Criteria

1. THE Platform SHALL authenticate Users using email and password credentials
2. WHEN a User logs in successfully, THE Platform SHALL create a session token valid for 24 hours
3. WHEN a User logs out, THE Platform SHALL invalidate the session token immediately
4. THE Platform SHALL assign one or more roles to each User
5. THE Platform SHALL enforce role-based permissions for all protected operations
6. WHEN a User attempts an unauthorized action, THE Platform SHALL return an error and log the attempt
7. THE Platform SHALL record all authentication events in Audit_Logs

### Requirement 2: Keyword and Keyword Group Management

**User Story:** As a monitoring analyst, I want to create keyword groups with keywords, negative keywords, and excluded keywords, so that I can precisely target relevant brand mentions and filter out noise.

#### Acceptance Criteria

1. THE Platform SHALL allow Users to create, update, and delete Keyword_Groups
2. THE Platform SHALL allow Users to add Keywords to Keyword_Groups
3. THE Platform SHALL allow Users to add Negative_Keywords to Keyword_Groups that exclude mentions when matched
4. THE Platform SHALL allow Users to add Excluded_Keywords to Keyword_Groups that filter out irrelevant mentions
5. THE Platform SHALL support AND, OR, and NOT logical operators within Keyword_Groups
6. THE Platform SHALL assign priority levels (1-5) to Keyword_Groups
7. THE Platform SHALL assign Alert_Threshold values (0-100) to Keyword_Groups
8. WHEN a Mention's Risk_Score exceeds the Keyword_Group Alert_Threshold, THE Platform SHALL create an Alert
9. THE Platform SHALL validate that each Keyword contains at least one non-whitespace character
10. WHEN a Keyword is updated, THE Platform SHALL apply changes to future Crawl_Jobs only

**User Story:** As a monitoring analyst, I want to monitor Facebook public pages and groups for brand mentions, so that I can track reputation on Vietnam's most popular social media platform.

#### Acceptance Criteria

1. THE Platform SHALL collect data ONLY from Facebook public pages and groups that are legally accessible without authentication

3. WHEN a Crawl_Job targets a Facebook Source, THE Platform SHALL collect public posts from the page or group

5. THE Platform SHALL extract post content, author name, published timestamp, like count, share count, and comment count
6. THE Platform SHALL extract comment content, author name, published timestamp, and like count
7. THE Platform SHALL respect Facebook API rate limits (200 calls per hour per user token)
8. WHEN Facebook API rate limit is exceeded, THE Platform SHALL pause requests and resume after the rate limit window resets
9. WHEN Facebook API returns an error, THE Platform SHALL log the error with error code and message
10. THE Platform SHALL store Facebook access tokens securely using encryption at rest
11. THE Platform SHALL validate Facebook access token validity before each Crawl_Job
12. WHEN a Facebook access token expires, THE Platform SHALL notify Admins via Alert
13. THE Platform SHALL comply with Facebook Platform Terms and Developer Policies at all times
14. THE Platform SHALL NOT use hacking, DDoS, spam, account takeover, captcha bypass, or unauthorized scraping techniques
